# Редизайн «Свадебная плёнка» — аналоговая плёночная тема

**Дата:** 2026-07-12
**Статус:** утверждено

## Цель

Довести метафору «одноразовая плёнка на 10 кадров» до конца визуально. Сейчас она живёт только в текстах и счётчике; после редизайна каждый экран строится вокруг неё. Настроение: тёплая светлая бумага + винный акцент, дневное свадебное, ностальгичное.

Меняются только представления (JSX/CSS). Логика — API, upload с ретраями, кэши, Embla, polling, admin — не трогается.

## Утверждённые приёмы

- Эффект «проявки» новых фото в галерее
- Перфорация плёнки и кассета как визуальные элементы
- «Живая галерея-стол»: лёгкие повороты карточек
- Без звука, без вибрации, без вспышки экрана
- Без тёмной темы / darkroom

## 1. Фундамент темы

**Типографика.** Три роли:
- Акцидентный serif с кириллицей — Cormorant (Google Fonts, есть кириллица) — заголовки, имена пары.
- Montserrat Alternates — текстовый (остаётся).
- JetBrains Mono — «технические» подписи в духе маркировки плёнки: `WEDDING 400 · 24.07.2026`, `FRAME 07`, датировки.

**Палитра.** Эволюция текущей:
- `--color-paper` — чуть теплее.
- Новые: `--color-sepia` (выцветший янтарный, для проявки и штампов даты), `--color-cream` (тёплые тени/подложки).
- `--color-wine`, `--color-ink`, `--color-line` остаются.
- Зерно фона: opacity с 0.04 до ~0.07, чуть крупнее.

**Перфорация.** Переиспользуемый элемент (CSS/SVG: ряд скруглённых прямоугольников). Используется как разделитель и рамка кадра — дозированно, не как сплошной фон.

## 2. Welcome-экран (NameForm)

Экран как «упаковка/коробка плёнки», сверху вниз:
1. Моно-маркировка: `WEDDING FILM · 10 EXP · 24.07.2026`.
2. Имена пары крупным акцидентным serif.
3. Вертикальная полоска плёнки с перфорацией — декоративная ось.
4. Правила игры (текущий текст «У тебя есть плёнка на 10 кадров…»).
5. Инпут имени + кнопка.

Элементы появляются каскадом (staggered) при заходе.

## 3. Экран камеры (CameraScreen)

- **Счётчик** — «окошко кассеты»: тёмная скруглённая плашка, внутри существующая анимация counter-roll, перфорация по краям.
- **Превью снимка** — кадр плёнки: тёмная рамка, перфорация сверху/снизу, моно-подпись `FRAME NN` (номер текущего кадра). Заменяет белый полароид.
- **Кнопка затвора** остаётся; добавляется кольцо-прогресс отснятых кадров вокруг неё.
- **Финал «Плёнка отснята»** — CSS/SVG-кассета с намотанной плёнкой вместо эмодзи 🎞.

## 4. Галерея

- **Повороты:** каждая карточка получает детерминированный поворот в диапазоне ±1.5°, вычисленный из hash(photo.id) — стабильный между рендерами и повторными заходами. Лёгкий разброс теней тем же способом.
- **Проявка:** фото, появившиеся после первого рендера в текущей вкладке, различаются через существующий модульный `photosCache` (id, которых не было в кэше на момент первого рендера). Новые входят с анимацией ~1.2 с: сепия + blur + пересвет → нормальное изображение. Старые рисуются сразу, как сейчас.
- **Подпись** под фото — моно-датировка: `АНЯ · 21:37` (имя капсом).
- **Заголовок** — serif, перфорированный разделитель под шапкой.

## 5. Лайтбокс

Минимальные правки: фото в рамке кадра с перфорацией сверху/снизу; подпись — «штамп даты» янтарным (sepia) моно-шрифтом, как выжженная дата на плёночных отпечатках.

## Доступность и ограничения

- `prefers-reduced-motion`: проявка вырождается в простой fade, каскады и повороты-анимации отключаются (сами статичные повороты остаются).
- Zero-CLS сохраняется: aspect-ratio-подход в галерее не меняется.
- Всё оформление — CSS/SVG инлайном, без новых зависимостей и без растровых ассетов.

## Вне скоупа

- Admin-страница (служебная, не для гостей).
- Звуки, haptics, вспышка, тёмная тема.
- Любые изменения API, схемы БД, логики загрузки.
